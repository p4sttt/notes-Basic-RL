# Event-loop в RL

- Начальное состояни
- Основное состояни
- Термиальное состояние

$E=\{ x=(x_{1}, \dots, x_{n})\ |\ x_{i} \in \mathbb{R}, i=\overline{1,n} \}$ — множество состояний
$A=\{ a:E\to E \}$ — множество действий

В ответ на действие система переходит в другое состояние и возвращает некоторую награду за действие

Выделяют два сценария взаимодействия агента со средой:

- конечные процессы
- бесконечные

# Формализация модели

Марковским процессом приятия решиний назовем упорядоченную четверку $(S, A, P, R)$, где
- $S$ — множество состоний
- $A$ — множество действий
- $P_{a}(s, s')$ — веротяность перехода из состояния $s$ в состояние $s'$ при совершении действия $a$
- $R_{a}(s, s')$ — мгновенная награда при переходе из состояния $s$ в состояние $s$ после совершения действия $a$

Не всега система описывается детерменированным набором вероятностей, поэтому можно ввести альтернативное определение, с упорядоченной тройкой $(S, A, R)$

$\mathcal{A} = \{ 1,\dots,k \}$ 
$\mu=\mathbb{E}[R_{a}(s, s')]$
$a^*=\arg \max_{a \in A} \mu_{a}$ — оптимальное действие для текущего состояния это такое действие, которое максимизирует матожидание награды

$H=\left( A_{1}, R_{1}, \dots, A_{n}, R_{n},\dots \right)$ — история совершенных действий и полученных за них наград. Первые $k$ шаг из истории будем обозначать $H_{k}$

$\pi: H_{k} \to \mathcal{A}$ — политика, определяющая действия, совершаемые в разных состояниях

$A_{t} \sim \pi_{t}$ — семплируем действия из политики
$\mathbb{E}_{\pi}[ \cdot]$ — матожидание при политике $\pi$

$S_{T} = \sum_{t=1}^T R_{t}$  — сумма наград (состояние в многоруком бандите)
$\mathbb{E}[S_{T}]$ ← хотим максимизировать награду

$\mathcal{R}(T)= \mathcal{T}_{\mu^*} - \mathbb{E}_{\pi}\left[  S_{T}\right]$ — начинаем работать с автоматом ($\mathcal{T}_{\mu^*}$ – модель с оптимальными параметрами), эта величина оценивает "полезность" каждого действия (ошибка, за первые $T$ тактов)

Проблема Exploration & Exploitation

Если модель постоянно будет действовать по жадному алгоритму, то будут возникать ситуации, когда модель не сможет находить новые состояния из-за чего будут не найдены глобально оптимальные алгоритмы. Для решения этой пробемы можно ввести шум в принятые действия (esp-задный алгоритм)
$a = \arg \max_{a \in A} \mu_{a}$ с вероятснотью $1-\epsilon$, и $a=\text{random} A$ с вероятснотью $\epsilon$
